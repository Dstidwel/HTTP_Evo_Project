HTTP/2 — Multiplexing & Compression

HTTP/2 introduces a binary framing layer and multiplexing multiple streams over one connection while using HPACK to compress repetitive headers.

Core Features

  • One TCP connection per origin with interleaved streams
  • Header compression reduces overhead
  • Server Push (experimental; deprecated in most browsers)

Tradeoffs

Still rides on TCP, so packet loss can impact all streams (transport-level HOL blocking).
