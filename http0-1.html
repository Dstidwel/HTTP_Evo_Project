HTTP/0.9 → HTTP/1.0

HTTP/0.9 was a single-command protocol primarily for fetching HTML. No headers, no status codes, and only raw document responses.

HTTP/1.0 (RFC 1945) generalized HTTP with headers, MIME types, and status codes, enabling images, styles, and rich media. Backward compatibility eased adoption.

Key Differences

AspectHTTP/0.9HTTP/1.0
Message FormatPlain HTML onlyHeaders + body
Status CodesNoneStandardized (e.g., 200, 404)
Media TypesHTMLMIME types (images, etc.)
Caching—Header-driven caching (basic)
ConnectionsOne request/connectionTypically non-persistent

Why the Change Mattered

  • Enabled rich media and better content negotiation
  • Introduced consistent semantics via status codes
  • Laid groundwork for caching and proxies
