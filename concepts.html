Key Concepts

Stateless
Each request is independent; servers don’t retain client state by default. State is layered via cookies, sessions, or tokens.
Headers
Metadata pairs (e.g., Content-Type, Cache-Control) that guide behavior for clients, servers, and intermediaries.
Status Codes
Standard outcomes like 200 OK, 301 Moved Permanently, 404 Not Found, 500 Internal Server Error.
Persistent Connections
Introduced in HTTP/1.1 to reuse TCP connections across multiple requests, reducing handshake overhead.
Multiplexing
HTTP/2 feature allowing multiple streams over one connection, improving throughput for resource-heavy pages.
QUIC
UDP-based transport used by HTTP/3 with built-in TLS 1.3, reducing latency and head-of-line blocking.
